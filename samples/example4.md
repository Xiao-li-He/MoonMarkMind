## **1. 基础启动与示例**

- 页面能正常加载，左右两栏都显示。
- 顶部“示例/片段”下拉能切换不同示例。
- 切换示例后：
  - 左侧 Markdown 更新。
  - 右侧脑图同步更新。
  - 选中状态、编辑状态、撤销栈不要残留到新示例。

## **2. Markdown 编辑器同步**

- 直接在左侧 Markdown 里修改标题文字，右侧脑图实时更新。
- 增加/删除标题行，脑图结构同步变化。
- 混合标题和列表，例如 `######` 后接列表，层级显示正确。
- 在 Markdown 编辑器里正常拖选文字、复制、粘贴。
- 点击脑图节点后，左侧对应行高亮/滚动定位正确。
- 左侧编辑器里的原生文字选择仍然可用；编辑器外不要出现蓝色原生选区。

## **3. 节点选择与编辑**

- 单击节点：节点被选中，工具栏按钮启用。
- 点击空白画布：取消节点选中。
- 双击节点标题：进入标题编辑。
- 标题编辑：
  - `Enter` 保存。
  - `Escape` 取消。
  - 失焦保存。
  - 空保存/重复保存不产生多余撤销记录。
- 选中一个节点后，再选另一个节点，选择边框和左侧行高亮都正确更新。

## **4. 节点增删改结构**

测试工具栏和快捷键两套入口：

- 新增同级分支。
- 新增子节点。
- 删除节点，确认整棵子树被删除。
- 上移/下移节点：
  - 同级节点顺序正确变化。
  - 子树跟随移动。
  - 不该跨父级时不要跨。
- 降级/升级层级：
  - `Tab` 新增子节点。
  - 工具栏“缩进/提升”改变层级。
  - 标题超过 6 层后转列表。
- 折叠/展开节点：
  - 子节点隐藏/显示。
  - 折叠后再编辑/移动/撤销，路径展开合理。

## **5. 鼠标拖动移动节点**

这是最近改动最多的部分，建议重点测。

- 在逻辑图布局下拖动节点：
  - 拖到目标上方显示“插入到上方”。
  - 拖到目标下方显示“插入到下方”。
  - 拖到目标中部表示“作为子节点”。
  - 蓝色投放区域高度为 8px，不遮挡当前节点。
  - 悬浮提示只在命中有效目标节点时出现。
- 松开后：
  - 成功移动：选中移动后的节点。
  - 无效移动/拖回原位：节点位置恢复，样式不变灰，仍选中刚刚拖动的节点。
  - `Escape` 取消拖动：恢复拖动前状态。
- 禁止场景：
  - 不能拖当前渲染根节点。
  - 不能拖 synthetic root。
  - 不能拖入自身子树。
  - 从按钮、正文、图片预览、编辑器区域按下，不应该启动拖动。
- 跨类型移动：
  - 标题拖到列表区域：整棵标题子树变成列表项，不出现标题把列表节点反过来“吃掉”的情况。
  - 列表拖到标题区域：保持当前转换为标题区域的行为。
  - 标题拖到标题区域：层级正确重写。
  - 列表拖到列表区域：缩进/marker 正常。

## **6. 三种布局与样式组合**

每个组合至少快速测一遍选择、拖动、折叠、连线：

- 布局：
  - 逻辑图
  - 思维图
  - 树状图
- 样式：
  - 填充
  - 线条
- 层级显示：
  - 精简
  - 中等
  - 全部
- 特别看：
  - 连线长度是否正常。
  - 节点没有明显偏移。
  - 折叠节点计数/按钮位置正常。
  - 树状图中拖动 before/after 的命中方向是否符合预期。
  - 切换布局/样式/层级显示后，不残留旧布局的连线或节点位置。
  - 当前选中或刚操作的节点不要跳出可视区；没有选中节点时根节点应保持在合理视野内。

## **7. 备注与正文块**

选中节点后测试“插入”下拉：

- 插入笔记：
  - 新节点没有正文时创建正文。
  - 已有正文时进入编辑。
  - 保存、取消、失焦保存正常。
- 插入图片：
  - 默认生成 Markdown 图片块。
  - 编辑 alt/url 后保存。
  - 图片预览能打开，关闭正常。
  - 不安全图片地址不要渲染成可执行内容。
- 插入代码：
  - 多行代码保存后换行保留。
  - 渲染为代码块。
- 插入表格：
  - 表格渲染正常。
  - 编辑保存后 Markdown 不乱。
- 正文块删除：
  - 只删除当前块。
  - 可撤销。

## **8. Markdown 渲染内容**

在节点标题和正文里测试：

- 粗体、斜体、删除线、行内代码。
- 链接。
- 图片。
- 任务项 `[ ]` / `[x]`。
- 表格。
- 代码块。
- LaTeX 数学：
  - 行内公式。
  - 块级公式。
  - 分数、根号、上下标、对齐行。
- HTML 注入文本应被转义，不应执行。

## **9. 撤销/重做**

分别测试工具栏按钮和快捷键：

- `Ctrl+Z`
- `Ctrl+Y`
- `Ctrl+Shift+Z`

覆盖操作：

- 改标题。
- 新增节点。
- 删除节点。
- 移动节点。
- 插入/编辑/删除正文块。
- 折叠状态下编辑后撤销，祖先路径应该展开，隐藏的后代不要乱展开。

## **10. 画布平移与缩放**

- 鼠标滚轮缩放。
- `+` / `-` 按钮缩放。
- 点击 `100%` 数字进入手动输入。
- 输入非法缩放值后能恢复。
- 适配视图。
- 居中视图。
- 初次打开页面和切换示例后，默认 `100%` 视图应居中显示主要节点。
- 如果渲染后脑图异常跑出可视区，下一帧应自动恢复到可见位置。
- 双击空白区域居中/重定位。
- 拖动画布空白区域平移。
- 在节点、按钮、编辑器上拖动时不要误触发画布平移。
- 缩放/平移后再移动节点、选择节点、导出，位置不要错乱。

## **11. 导入导出**

- 下载 Markdown：
  - 文件名正确。
  - 内容和当前编辑器一致。
- 上传 Markdown：
  - 文件名 badge 更新。
  - 内容解析正确。
  - 历史状态清空或符合预期。
- 导出 PNG：
  - 能下载。
  - 节点、连线、正文、图片显示完整。
  - 当前画布缩放/平移不应该污染导出结果。
  - 导出完成后，当前页面的缩放、平移、选中状态保持不变。
- 导出 SVG：
  - 能下载。
  - SVG 打开后连线和节点位置正确。
  - 标题或正文里的 `<`, `>`, `&`, 引号等特殊字符被正确转义，不破坏 SVG/XML。
- 导出 HTML：
  - 打开后静态脑图能正常显示。
  - 备注/图片/连线样式正常。
  - 不依赖当前编辑页面状态。
  - 导出的 HTML 文件可独立打开验证，不依赖当前编辑页的缩放、平移、选中状态。

## **12. 响应式和边界情况**

- 调整浏览器窗口大小。
- 拖动左右分栏分隔条。
- 很长标题自动换行，不溢出按钮/节点。
- 很深层级结构可滚动、可拖动。
- 空文档/只有正文/没有标题的文档。
- 只有一个根标题的文档。
- 大文档下切换布局和导出不要明显卡死。
- 应用界面普通按钮、标签、画布区域不再出现蓝色原生文本选区。

## **13. 7 月更新重点回归**

以下内容来自 2026 年 7 月提交记录，重点覆盖网页端、Markdown 渲染、脑图渲染、导出 runtime 和连接线。

### Markdown 折叠预览

- 折叠只有标题、没有正文和子节点的空标题：
  - 左侧行号不漂移。
  - 当前行高亮仍对应原始标题行。
  - 右侧预览不出现多余空白占位。
  - 展开后空标题仍在原位置。
- 折叠包含正文、子标题、列表、代码块的标题：
  - 折叠后只隐藏该标题的后代内容。
  - 折叠提示/图标位置和样式正常。
  - 展开后正文块、子标题、列表、代码块完整恢复。
- 折叠后点击脑图节点或左侧可见行：
  - 左侧滚动定位到正确可见行。
  - 隐藏行不会被错误选中或高亮。
  - 编辑可见标题后，折叠状态不应导致行号错位。

### Markdown 渲染内容补测

- 在正文里输入普通换行和列表混排：
  - 普通换行按 Markdown 规则显示。
  - 有序/无序列表缩进正确。
  - 非规范列表 marker 作为正文显示，不误解析成列表。
- 输入转义 Markdown 标记，例如 `\*not italic\*`、`\# not heading`：
  - 页面显示字面量标记。
  - 不应被渲染成斜体或标题。
- 输入高亮、代码、表格、任务项和 LaTeX：
  - 高亮样式、代码块背景、表格边框、任务项勾选状态、公式样式都正常。
  - 行内公式和块级公式的字号、行距不挤压相邻内容。
- 输入 HTML 注入文本，例如 `<script>alert(1)</script>`：
  - 应作为安全文本显示或被转义。
  - 不弹窗，不执行脚本，不破坏后续 Markdown 渲染。

### 脑图可见性和视口保持

- 首次打开页面：
  - 脑图以 `100%` 显示。
  - 根节点和主要第一层分支位于可视区域内。
  - 连接线已绘制，不需要手动缩放或拖动才能看到内容。
- 切换示例、上传 Markdown、清空后新建：
  - 新文档内容进入可视区域。
  - 旧文档的平移、缩放、选中节点、折叠状态不污染新文档。
- 选中一个较深层节点后切换布局、样式、层级显示：
  - 该节点优先保持可见。
  - 若节点因层级显示被隐藏，则根节点或可见祖先应回到合理视野。
  - 画布不要突然跳到大空白区域。

### 布局与节点间距

- 逻辑图：
  - 子节点整体在根节点右侧排列。
  - 深层节点之间有足够间距，标题和正文不重叠。
  - 单子节点链条的连线方向稳定，不出现折返。
- 思维图：
  - 根节点左右分支分布合理。
  - 左右两侧连接线分别接到节点对应侧边。
  - 中心线/线条风格节点的横线样式清晰，和连接线对齐。
- 树状图：
  - 父节点在上，子节点在下。
  - 折叠按钮、底部操作按钮和连线位置不互相遮挡。
  - 多个兄弟节点间距足够，长标题换行后仍不重叠。
- 使用很长标题、很多正文块、图片节点、深层级文档重复上述布局切换：
  - 节点间距随内容高度扩展。
  - 连线端点仍贴近父子节点，不穿过标题正文。

### 连接线层回归

- 缩放、平移、拖动画布后观察连接线：
  - 连接线跟随节点位置移动。
  - 线宽视觉稳定，不因缩放变得过粗或过细。
  - 不出现上一帧残留线、断线或重复线。
- 折叠/展开节点、编辑标题、编辑正文、拖动节点、调整浏览器窗口：
  - 连接线在操作后自动重绘。
  - 隐藏节点的连接线同步隐藏。
  - 展开后连接线恢复到正确父子关系。
- 点击节点、拖动节点、点击工具按钮、打开图片预览：
  - 连接线层不拦截鼠标事件。
  - 节点选中、拖动、按钮点击和图片预览都能正常触发。
  - 连接线不盖住节点文字和操作按钮。

### SVG/PNG/HTML 导出回归

- 在缩放、平移、选中节点、折叠部分节点后分别导出 PNG、SVG、HTML：
  - 导出文件显示完整脑图，而不是当前被平移裁掉的视口。
  - 节点、正文、备注、图片、折叠状态和连接线都完整。
  - 导出完成后当前页面视图状态不变。
- SVG 导出：
  - 浏览器能直接打开。
  - 特殊字符、中文、数学公式、代码块不会破坏 XML。
  - 连接线位置和网页端一致。
- PNG 导出：
  - 图片边界留白合理。
  - 大图不会明显模糊。
  - 能显示可内联的图片；不能内联的外部图片不应导致整张导出失败。
- HTML 导出：
  - 文件可独立打开。
  - 样式、连接线、备注、图片预览不依赖当前编辑页面。
  - 打开后不需要联网也能显示基础脑图样式。

### 导出 HTML runtime

- 打开导出的 HTML 后点击折叠/展开按钮：
  - 子树隐藏/显示正确。
  - 折叠按钮状态同步变化。
  - 连接线同步重排。
- 打开导出的 HTML 后展开/收起备注：
  - 备注正文显示/隐藏正确。
  - 节点高度变化后连接线重新对齐。
- 在导出的 HTML 中点击图片预览：
  - 预览层能打开。
  - 点击关闭或遮罩能关闭。
  - 关闭后页面仍可继续折叠/展开。
- 调整导出 HTML 的浏览器窗口大小：
  - 节点位置和连接线随窗口变化重新计算。
  - 不出现连线偏移到旧位置的问题。
